# CAVE Infrastructure Docker

A Docker deployment wrapper for the [CAVE Infrastructure](https://gitlab.opencode.de/BSI-Bund/cave/cave-infrastructure-docker) project by BSI-Bund.

This repository provides an easy way to build and run CAVE Infrastructure in Docker containers, including all necessary dependencies (Python, OpenTofu, OpenStack CLI, WireGuard).

## Prerequisites

- Docker and Docker Compose installed
- OpenStack credentials (RC file or environment variables)
- SSH key for deployment
- (Optional) GitHub Container Registry access for pushing images

## Setup

### 1. OpenStack
You will need a running OpenStack environment. If you need to set one up, MicroStack (Canonical OpenStack) is a good starting point: [Canonical OpenStack Documentation](https://canonical-openstack.readthedocs-hosted.com/en/latest/)

### 2. Configuration
Clone this repository and create the configuration files:

```bash
git clone https://github.com/FelixHertweck/cave-infrastructure-docker.git
cd cave-infrastructure-docker

# Create your .env file
cp .env.sample .env

# (Optional) Copy your OpenStack RC file if you use one
cp .openrc.sample .openrc
```

Edit the `.env` file to set your credentials, lab prefix, SSH key name, and network configurations. Ensure your SSH key is located in the `./ssh-keys/` directory.

> **Note on `.openrc`**: If you use an `.openrc` file generated by the OpenStack Dashboard, you **must remove the interactive password prompt** (`read -sr OS_PASSWORD_INPUT`) and set `export OS_PASSWORD=your-actual-password-here` directly.

### 3. Post OpenStack Initialization
If your OpenStack VMs cannot reach the internet, you likely need NAT for the external OpenStack network on your host machine. Run the initialization script on the host (requires root privileges):

```bash
sudo ./scripts/post-openstack-init.sh
```

### 4. Build CAVE Base Images
Before deploying, you must build the required base images and upload them to OpenStack using Packer. By default, this uses the official [CAVE Images repository](https://gitlab.opencode.de/BSI-Bund/cave/cave-images).

Start the interactive image builder:
```bash
docker compose run --rm cave /cave/build-images.sh
```

You can also specify a custom repository and a specific branch or commit hash:
```bash
# Example: Using a custom repo and the 'my-feature-branch' branch
docker compose run --rm cave /cave/build-images.sh https://git.example.com/custom-images.git my-feature-branch
```

The standard image repository expects a set of mapped OpenStack flavors. The default mapping in this repo translates the upstream image names to the flavors used by your cloud.

Typical flavor mapping examples:
- `client-medium`
- `server-small`
- `server-large`
- `server-windows`
- `m1.small`

To inspect available flavors in your OpenStack environment, run:
```bash
docker compose run --rm cave openstack flavor list
```

If the flavor names differ from the standard repository, add the correct mapping to your `.env` file using `OS_FLAVOR_MAP`.

Follow the interactive prompts to build the necessary images. Wait until the builds finish successfully.

### 5. Deploy Infrastructure using the Wrapper
Now you can deploy the CAVE infrastructure using the interactive deployment wrapper.

Start the wrapper interactively:
```bash
docker compose run --rm cave /cave/deploy-wrapper.sh
```

The wrapper will:
- Validate your credentials and SSH key
- Let you choose an available configuration file (`/backend/configs/`)
- Automatically detect the associated user configuration file
- Support deploying with WireGuard or OpenVPN

Alternatively, you can skip the interactive menu and deploy directly:
```bash
# Deploy with OpenVPN
docker compose run --rm cave /cave/deploy-wrapper.sh day1

# Deploy with WireGuard and a custom prefix
docker compose run --rm cave /cave/deploy-wrapper.sh day1 --wg --lab-prefix my-training
```

## Troubleshooting

### ❌ "Please enter your OpenStack Password..." prompt appears

**Problem**: Your `.openrc` file contains the interactive password prompt.

**Solution**: Edit `.openrc` and replace the interactive `read -sr` lines with:
```bash
export OS_PASSWORD=your-actual-password-here
```

### ❌ "OS_PASSWORD is not set" error

**Problem**: Both `.env` and `.openrc` are missing the password.

**Solution**: 
Ensure `.env` has `OS_PASSWORD=your-password` or `.openrc` has `export OS_PASSWORD=your-password`.

## Manually Push to GitHub Container Registry

```bash
# Build locally (with BuildKit for better caching)
DOCKER_BUILDKIT=1 docker compose build

# Tag and push a version
docker tag ghcr.io/felixhertweck/cave-infrastructure-docker:latest ghcr.io/felixhertweck/cave-infrastructure-docker:v1.0.0
docker push ghcr.io/felixhertweck/cave-infrastructure-docker:v1.0.0
```
